Homebrew 5.1.0 has been released.
Homebrew's most significant changes since 5.0.0 are expanded `brew bundle` support, `brew version-install`, new `-full` formula handling and installer updates.

Major changes and deprecations since 5.0.0:

- [`brew bundle` manages Rust Cargo packages from `Brewfile`s](https://github.com/Homebrew/brew/pull/21250), [`uv` tools](https://github.com/Homebrew/brew/pull/21614) and [Flatpak remotes or apps](https://github.com/Homebrew/brew/pull/21097).
- [`brew bundle` fetches downloads in parallel before install](https://github.com/Homebrew/brew/pull/21252), [auto-taps formulae and casks from external taps](https://github.com/Homebrew/brew/pull/21654), [uses `mas get` for better fresh Apple ID handling](https://github.com/Homebrew/brew/pull/21590), [sets `HOMEBREW_INSIDE_BUNDLE`](https://github.com/Homebrew/brew/pull/21360) and [adds `--no-secrets` for safer environment inspection](https://github.com/Homebrew/brew/pull/21137).
- [`brew version-install` creates, extracts and installs older formula versions in one step.](https://github.com/Homebrew/brew/pull/21418)
- [Homebrew publishes an unversioned `Homebrew.pkg` installer URL](https://github.com/Homebrew/brew/pull/21448) and [uses Portable Ruby 4.0.1 with Ruby 4.0](https://github.com/Homebrew/brew/pull/21459).
- [Homebrew adds `ffmpeg-full`](https://github.com/Homebrew/homebrew-core/pull/261303), [`imagemagick-full`](https://github.com/Homebrew/homebrew-core/pull/263377), [automatic linking for versioned or `-full` keg-only formulae when the unversioned sibling is absent](https://github.com/Homebrew/brew/pull/21676) and [stricter rejection of dependencies on `*-full` formulae](https://github.com/Homebrew/brew/pull/21471).
- [Homebrew deprecates `Formula#needs` and `needs :openmp`](https://github.com/Homebrew/brew/pull/21169), [`option :cxx11` and `option :universal`](https://github.com/Homebrew/brew/pull/21181) and [other APIs scheduled for the normal release-cycle removals](https://github.com/Homebrew/brew/pull/21629).

Upcoming changes:

- [Homebrew plans the Ubuntu 24.04 CI migration for 5.2.0, which is expected to raise the Linux baseline to `glibc` 2.39 and `libstdc++` 6.0.33.](https://github.com/Homebrew/brew/pull/21613#issuecomment-3946063773)
- [Homebrew continues the `master` to `main` migration introduced in 4.6.0; some repositories stop updating `master` in 5.1.0](https://github.com/Homebrew/actions/pull/702) and [the remaining compatibility branches are scheduled for removal in 5.2.0 in favour of `main`](https://github.com/Homebrew/actions/pull/704).

Homebrew's other changes since 5.0.0 worth highlighting are the following:

- [`brew services` supports `nice` priorities for services.](https://github.com/Homebrew/brew/pull/21088)
- [`brew services` sets `ThrottleInterval` for LaunchAgents and LaunchDaemons on macOS.](https://github.com/Homebrew/brew/pull/21095)
- [`brew sh --ruby` opens a shell with Homebrew's bundled Ruby and Bundler preconfigured.](https://github.com/Homebrew/brew/pull/21223)
- [`brew shellenv` uses simpler shell detection and is easier to override when auto-detection is wrong.](https://github.com/Homebrew/brew/pull/21374)
- [`brew lgtm` is recommended in the PR template before submitting changes](https://github.com/Homebrew/brew/pull/21033) and [`brew lgtm --online` covers checks that need network access](https://github.com/Homebrew/brew/pull/21036).
- [`brew --taps` prints Homebrew's taps directory.](https://github.com/Homebrew/brew/pull/21058)
- [`brew source` jumps straight to a formula's source repository.](https://github.com/Homebrew/brew/pull/21339)
- [`brew tap-info` exits with a failure status for unknown taps, which makes scripting more reliable.](https://github.com/Homebrew/brew/pull/21310)
- [`brew edit` auto-taps Homebrew/core or Homebrew/cask when API-known formulae or casks are edited on tapless installs.](https://github.com/Homebrew/brew/pull/21641)
- [`brew config` on macOS reports the detected Metal toolchain version.](https://github.com/Homebrew/brew/pull/21309)
- [`brew info` marks installed and uninstalled formulae or casks more clearly.](https://github.com/Homebrew/brew/pull/21386)
- [`brew search` labels deprecated and disabled formulae and casks directly in search results.](https://github.com/Homebrew/brew/pull/21516)
- [`brew outdated` respects `HOMEBREW_UPGRADE_GREEDY_CASKS`.](https://github.com/Homebrew/brew/pull/21138)
- [`brew doctor` warns early when Linux `glibc` is older than the upcoming CI baseline.](https://github.com/Homebrew/brew/pull/21616)
- [`brew uninstall` refuses to uninstall casks that other installed casks depend on](https://github.com/Homebrew/brew/pull/21078) and [keeps going when one requested formula or cask fails](https://github.com/Homebrew/brew/pull/21617).
- [`brew upgrade` and `brew reinstall` keep working through individual name-resolution failures instead of stopping early.](https://github.com/Homebrew/brew/pull/21618)
- [Homebrew requires AI-assistance disclosure in PRs](https://github.com/Homebrew/brew/pull/21510), [documents preferred AI and LLM usage in contributions](https://github.com/Homebrew/brew/pull/21415) and [ships a `CLAUDE.md` guide alongside `AGENTS.md`](https://github.com/Homebrew/brew/pull/21504).
- [Homebrew GitHub Actions include `cache-homebrew-prefix` for prefix caching in CI.](https://github.com/Homebrew/actions/pull/786).
- [Homebrew GitHub Actions include `setup-ruby` for Ruby setup through Homebrew in CI.](https://github.com/Homebrew/actions/pull/787)
- [Homebrew preserves `WSL_DISTRO_NAME`, which keeps CLI tools launched from WSL behaving correctly.](https://github.com/Homebrew/brew/pull/21047)
- [Homebrew shows a progress bar for concurrent downloads.](https://github.com/Homebrew/brew/pull/21183)
- [Homebrew audits better distinguish true self-submissions from other new-formula submissions.](https://github.com/Homebrew/brew/pull/21512)
- [Homebrew allows dashes anywhere in tap names, not just after the `homebrew-` prefix.](https://github.com/Homebrew/brew/pull/21414)
- [Homebrew auto-bumps resources within formulae that have explicit `livecheck` blocks.](https://github.com/Homebrew/brew/pull/21623)
- [Homebrew points issue reporters to GitHub Discussions sooner when issues are not the best fit](https://github.com/Homebrew/brew/pull/21068) and [updates contributor guidance on when to open discussions, PRs and issues](https://github.com/Homebrew/brew/pull/21391).
- [Homebrew checks SSSE3 support more clearly on Linux `x86_64` when Homebrew requires it.](https://github.com/Homebrew/brew/pull/21178)
- [Homebrew falls back to simpler output on dumb terminals, which avoids garbled progress display in shells such as Emacs.](https://github.com/Homebrew/brew/pull/21659)
- [Homebrew keeps custom `HOMEBREW_CURLRC` paths through `brew update`.](https://github.com/Homebrew/brew/pull/21462)
- [Homebrew points unsupported older macOS releases to MacPorts instead of unsupported Homebrew installs.](https://github.com/Homebrew/brew/pull/21054)
- [Homebrew removes legacy migration code for the old `linuxbrew` repository](https://github.com/Homebrew/brew/pull/21200), [removes the remaining automatic migration logic for old `linuxbrew-core` formulae](https://github.com/Homebrew/brew/pull/21195) and [removes more obsolete `linuxbrew-core` migration entries](https://github.com/Homebrew/brew/pull/21190).
- [Homebrew uses the correct Portable Ruby URLs for custom `HOMEBREW_ARTIFACT_DOMAIN` mirrors](https://github.com/Homebrew/brew/pull/21075) and [uses the correct mirror links when building brew mirror URLs](https://github.com/Homebrew/brew/pull/21103).
- [Homebrew guides contributors toward the modern command API as more legacy CLI parser fallback paths are removed.](https://github.com/Homebrew/brew/pull/21546)
- [Homebrew contributor guidance tells LLM-assisted contributors to run the repository's own `./bin/brew` so local checks match CI more closely.](https://github.com/Homebrew/brew/pull/21628)
- [Homebrew reuses cached API data for up to 7 days in more places, reducing needless refreshes.](https://github.com/Homebrew/brew/pull/21262)
- [Homebrew lets old installed casks that still contain removed DSL such as `appcast` uninstall or upgrade again](https://github.com/Homebrew/brew/pull/21624) and [filters broken `Caskroom` metadata out of installed-cask listings](https://github.com/Homebrew/brew/pull/21630).
- [Homebrew closes cleanup lockfiles properly, avoiding a file descriptor leak.](https://github.com/Homebrew/brew/pull/21611)
- [Homebrew caching-proxy docs cover the Formulae API as well as bottles.](https://github.com/Homebrew/brew/pull/21212)
- [Homebrew cask acceptance docs explain the notability bar for self-submitted casks more clearly](https://github.com/Homebrew/brew/pull/21612) and [state that dropping support for current macOS versions can lead to rejection](https://github.com/Homebrew/brew/pull/21336).
- [Homebrew support-tier docs state that outdated macOS developer tools count as Tier 2](https://github.com/Homebrew/brew/pull/21710) and [`brew doctor` uses clearer wording about when to report issues from Tier 2 configurations](https://github.com/Homebrew/brew/pull/21711).
- [Homebrew formulae can use simpler shell-completion helpers, including a predefined `:cobra` format](https://github.com/Homebrew/brew/pull/21293) and [a built-in `:typer` format](https://github.com/Homebrew/brew/pull/21287).
- [Homebrew audits `revision` and `compatibility_version` changes more strictly, which helps catch ABI mistakes earlier.](https://github.com/Homebrew/brew/pull/20936)
- [Homebrew linkage checks fail indirect linkage problems more consistently, which helps catch broken dependencies earlier.](https://github.com/Homebrew/brew/pull/21407)
- [Homebrew `std_npm_args` disables package scripts by default, reducing exposure to malicious npm install hooks.](https://github.com/Homebrew/brew/pull/21136)
- [Homebrew container registry users can omit the Docker authentication header by setting `HOMEBREW_DOCKER_REGISTRY_BASIC_AUTH_TOKEN=none`.](https://github.com/Homebrew/brew/pull/21278)

Finally:

- [Homebrew simplified its governance model](https://github.com/Homebrew/brew/pull/21156), [cleaned up the related docs](https://github.com/Homebrew/brew/pull/21420) and [updated references to the current Lead Maintainers](https://github.com/Homebrew/brew/pull/21159).
- [Homebrew is running a short user survey to help inform future development.](https://github.com/orgs/Homebrew/discussions/6706)
- Homebrew accepts donations through [GitHub Sponsors](https://github.com/sponsors/Homebrew), [OpenCollective](https://opencollective.com/homebrew#category-CONTRIBUTE) and [Patreon](https://www.patreon.com/homebrew).

Homebrew thanks all our hard-working maintainers, contributors, sponsors and supporters for getting us this far.
